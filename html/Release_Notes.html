Hydrogeological Toolbox

Release Notes

Version 2.06

November 2017

This version is the one corresponding to the paper "Hytool: an open source matlab toolbox for the interpretation of hydraulic tests using analytical solutions" accepted for publication in the Journal of Open Source Software. Note that the license has been changed to MIT license. A few minor issues have been corrected. In particular some Community guidelines have been added on the GitHub page of the project.

Version 2.05

June 2017

This version contains a rather long series of minor bug fixes and adaptation to the release 2017 of matlab. The version has now been transfered to github.

Version 2.0

October 2007

This version corresponds to a major update of hytool. This was made possible thanks to the support of the Spanish Geological Survey (IGME) and the work of Damien Glenz and Sébastien Deyres.

Summary of the changes.

  • A systematic html help is provided, with search tools.
  • Demos have been prepared for most of the models.
  • The syntax of basic functions has changed in order to be more systematic.
    • The non linear fit function that were specific to each model (ths_fit, pcw_fit, csl_fit, etc.) are now replaced by a unique and generic fit function.
    • The functions allowing to plot the results of a test have named with the rpt suffix instead of tst suffix. Example: ths_rpt, etc.
    • The function slug_trial has been removed. The function trial can now be used both for pumping, slug or pulse tests.
    • The trial function now uses only the basename of a model. See: trial
    • A large number of functions under development has been removed from the main distribution. These functions are still available in the dev directory, but they are not systematically documented, and validated and should therefore be used with caution.
  • The Agarwal time formula has been coded for variable rate tests. See: agarwal_time
  • The report functions have been changed to allow interactive modification of the graphics.
  • Most of the routines have been improved and rewritten, in order to be more robust to errors and to provide a more systematic access to the toolbox fonctionalities. A large number of new options have been added.
  • Internal functions to build the help files were developped.

To know the history of the releases of Hytool, see the Version History document.
