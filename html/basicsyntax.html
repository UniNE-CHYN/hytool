Syntax convention

There are essentially two sets of functions available in hytool: the model specific functions and the general functions.

Model specific functions

The same functionality needs to be proposed for most of the mathematical models (Theis, Hantush, Gringarten, etc.). Therefore, to facilitate the identification of the functions we adopt function names that are made of a suffix and a prefix.

The prefix is a set of three letters that represent a certain model. For example, the Theis (1935) model is identified by the letters ths, the Papadopulos Cooper (1967) model for large diameter well is identified by pcw. See the list of models.

The suffix is also a set of three letters that indicates what the function does. For example, the function that computes the drawdown is noted dim, and therefore the function that computes the Theis model is ths_dim, the one that computes the Papadopulus Cooper model is pcw_dim. See the list of suffix to know the correspondence between the names of the functionalities.

General functions

They do not depend on a specific model. Their name indicates what the function does. For example, the function diagnostic creates a diagnostic plot.
